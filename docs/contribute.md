# Contribute

The tracker is open. Three ways to help.

## 1. Comment

Use the comments block at the bottom of any page to flag something we missed. Comments require a GitHub account. We monitor them and convert anything substantial into an issue.

## 2. Open an issue

Use a template. Each one asks for the source and the date so we can act quickly.

- [Propose a status change](https://github.com/L3-iGrant/eudi-wallet-tracker/issues/new?template=status-change.yml)
- [Add or update a source](https://github.com/L3-iGrant/eudi-wallet-tracker/issues/new?template=source-addition.yml)
- [Report a correction](https://github.com/L3-iGrant/eudi-wallet-tracker/issues/new?template=correction.yml)

## 3. Open a pull request

Every page has an **Edit this page** link in the footer. Clicking it opens GitHub's web editor, where you can change the underlying MDX or `data/eudi-status.json` directly and submit a pull request. Maintainers review every PR against the [methodology](/methodology).

## What we expect

- A source URL we can verify
- A date in `YYYY-MM-DD` format
- A one-line summary of what changed and why
- House style: British English, no em or en dashes, concise prose

## What happens next

1. A maintainer assesses the source against the evidence bar in the methodology.
2. If accepted, a new history entry is appended to the country's record. Nothing is deleted.
3. The site rebuilds, the SVG badge updates, and the country page reflects the change.

## Maintainer commitments

- New issues acknowledged within five working days.
- Monthly review issue opened on the 1st of each month, ordered by oldest history entry.
- All accepted sources snapshotted to the Wayback Machine so citations survive link rot.
